# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

`go-rag-arxiv` is a Go service that fetches ArXiv academic papers on a daily schedule and sends notifications via a Telegram bot.

## Commands

```bash
go run ./main          # Run the application
go build -o app ./main # Build executable
go test ./...          # Run all tests
go fmt ./...           # Format code
go vet ./...           # Static analysis

buf lint               # Lint proto files
buf generate           # Regenerate gRPC stubs into internal/gen/
```

## Architecture

```
main/main.go          entry point, signal handling, graceful shutdown
internal/
  app/                wires all components, owns the gocron scheduler
  client/arxiv/       ArXiv Atom feed HTTP client (rate-limited, PDF download)
  client/telegram/    Telegram Bot API client (SendMessage / SendHTML / SendMarkdown)
  cron/               scheduled jobs
    arxiv_fetcher.go  fetches last 24 h of papers per topic, sends HTML notifications
  wrappers/
    ratelimit.go      generic rate-limiter — wraps any func(ctx) error via Do()
  gen/arxiv/v1/       gRPC stubs generated by buf (do not edit)
proto/arxiv/v1/       protobuf source for ArxivService (Search, Ask RPCs)
```

### Key conventions

- **Functional options** (`Option func(*Config)`) for all client constructors.
- **Interfaces defined at point of use** — `fetcher` and `notifier` are declared inside the `cron` package, not in the client packages.
- **Rate limiting** via `wrappers.RateLimiter.Do(ctx, fn)` — wraps any callback; used in `ArxivFetcher` to throttle Telegram sends.
- `arxiv.Paper.PublishedAt` is `time.Time` (parsed from RFC3339 at the API boundary).
- Proto stubs live in `internal/gen/`; regenerate with `buf generate` whenever `.proto` files change.
